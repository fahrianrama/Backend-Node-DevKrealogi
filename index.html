Krealogi -

Overview

Backend-Krealogi adalah aplikasi interface (API) yang dapat digunakan untuk mengolah data yang berasal dari database krealogi dan diteruskan kepada client-side.

Documentation

Users

Users

Method : GET
Response : All Users (Json)
Link dapat diakses pada /api/users dengan response berupa data JSON dari tabel Users

Show User Data

Method : GET
Required : User ID
Response : User Data (Json)
Link dapat diakses pada /api/users/{id} dengan response berupa data JSON dari tabel Users

Auth

Method : POST
Required : Email, Password (POST)
Response : Auth Status & User Data (Json)
Link dapat diakses pada /api/auth dengan response berupa status login/auth dan data user
Note : Hanya dapat diakses dengan metode POST -> Ex:Postman

Logout

Method : GET
Response : Logout Status (Json)
Link dapat diakses pada /api/users/logout dengan response berupa status logout

Get ID

Method : GET
Response : ID (Json)
Link dapat diakses pada /api/getid dengan response berupa id user saat ini
Note : Data (null) apabila user belum login/auth

Store

Method : POST
Required : Email, Password, Role (POST)
Response : Store Status & Stored Data (Json)
Link dapat diakses pada /api/store dengan response status store data dan data yang dimasukkan
Note : Hanya dapat diakses dengan metode POST -> Ex:Postman

Verify

Method : POST
Required : Email, Verification Code (POST)
Response : Verify Status (Json)
Link dapat diakses pada /api/verify/{id} dengan response status verify data menjadi verified dan data User terbaru
Note : Hanya dapat diakses dengan metode POST -> Ex:Postman

Update

Method : POST
Required : Email, Password, Role, Status (Any) (POST)
Response : Update Status & Updated Data (Json)
Link dapat diakses pada /api/update/{id} dengan response status update data dan data yang dimasukkan
Note : Hanya dapat diakses dengan metode POST -> Ex:Postman

Delete

Method : GET
Required : User ID
Response : Delete Status (Json)
Link dapat diakses pada /api/delete/{id} dengan response status delete data
